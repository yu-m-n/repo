# 🤖 AI Agent 작업 규칙

당신은 **test-user** 의 개인 프로젝트 디렉토리에서 작업하는 AI 코딩 에이전트입니다.

## ⚠️ 절대 규칙 — 반드시 준수

### 1. 디렉토리 제한
- **수정 가능**: `members/test-user/` 내부 파일만
- **수정 금지**: 그 외 모든 경로 (다른 멤버 디렉토리, 루트 파일, `.github/`, `integrated/`, `docs/` 등)
- 다른 멤버의 코드를 참조해야 할 경우 **읽기만 가능**, 절대 수정하지 마세요

### 2. DEVLOG.md 업데이트 의무
작업을 수행할 때마다 반드시 `members/test-user/DEVLOG.md` 에 아래 형식으로 기록하세요:

```markdown
## [YYYY-MM-DD]
### 작업자
- 👤 본인 / 🤖 Agent / 👥 팀원(아이디) 도움

### 변경 요약
- (무엇을 했는지 한 줄 요약)

### 수정된 파일
- `파일경로` — 변경 설명

### 도움받은 내용 (해당 시)
- 누구에게, 어떤 부분을 도움받았는지 기록

### 상태: ✅ 완료 / 🔄 진행중
```

#### 작업자 표기 규칙
- Agent가 직접 작업: `🤖 Agent`
- 사용자가 직접 코딩 후 기록 요청: `👤 본인`
- 다른 팀원 도움: `👥 팀원(@아이디) 도움` + "도움받은 내용" 섹션에 상세 기록

### 3. 파일 생성 규칙
- 새 파일은 반드시 `members/test-user/` 하위에만 생성
- `src/` 디렉토리에 소스 코드 작성
- 설정 파일은 `members/test-user/` 루트에 배치

### 4. 환경 관련
- Python 패키지 추가 시 `environment.yml` 업데이트를 안내하세요
- conda 환경 이름: `test-user-env`

## 작업 스타일
- 한국어로 소통하세요
- 코드 주석은 한국어 또는 영어 모두 가능
- 변경 전 현재 코드를 먼저 확인하세요
- 큰 변경은 단계별로 진행하세요
